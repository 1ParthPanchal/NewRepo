We need to manage a user module. Which contains basic user information. Password should be stored in the encrypted format. Each user will be assigned to specific role. Role can be Basic and Admin and in future roles can be increased. Basic User can edit his/her personal information. Admin user can create, update and delete the information for any user.

Task1: 

  • Design the tables based on above use case and insert dummy entries in that. 
  • Use DBFirst Approach and generate a model and DB Context for the same. 

Task 2

  • Create the Login POST API.  Which accept a Username and Password and verify a user against a DB and its role. Create a token. And returns a proper status code, message and token to the user.

Task 3

  • Create the GET/PUT/POST/DELETE API for User.
    • Note:
      • Validation should be there at server side.
      • Accept the token in header based on the user role access should be given.
      • Basic user-> can do crud on its own details
      • Admin User-> can do crud on any user.
